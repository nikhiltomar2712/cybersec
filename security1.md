# 🔒 Security Policy

## Supported Versions

This is a **personal cybersecurity learning and notes** repository.
There are no software releases or versioned builds to patch.
The security policy applies to the **repository contents** itself.

| Content Type | Covered by This Policy |
|---|:---:|
| Accidentally committed credentials / tokens | ✅ |
| Malicious or unsafe code in any file | ✅ |
| Broken links redirecting to harmful sites | ✅ |
| Privacy leaks (PII, internal data) | ✅ |
| Incorrect notes or typos | ❌ (use Issues) |
| Feature suggestions | ❌ (use Issues) |

---

## 📬 Reporting a Vulnerability

> **Do NOT open a public GitHub Issue for security vulnerabilities.**  
> Public disclosure before a fix is in place puts everyone at risk.

### 🥇 Preferred — GitHub Private Security Advisory

1. Go to the **[Security tab](../../security)** of this repository
2. Click **"Report a vulnerability"**
3. Fill in the template and submit

GitHub keeps the report fully confidential until resolved.

### 🥈 Alternative — Email

```
To:      nikhiltomar2712@gmail.com
Subject: [SECURITY] <short description>
```

---

## 📋 What to Include

A clear report helps resolve issues faster. Please provide:

| Field | Details |
|---|---|
| **Summary** | What is the issue in one sentence? |
| **Location** | File path, line number, or URL |
| **Steps to Reproduce** | How can the issue be confirmed? |
| **Impact** | What could happen if unaddressed? |
| **Evidence** | Screenshot, log snippet, or code block |
| **Suggested Fix** | *(Optional)* Your recommendation |

---

## ⏱️ Response Timeline

| Action | Target |
|---|---|
| Acknowledge receipt | **≤ 48 hours** |
| Confirm / triage issue | **≤ 5 days** |
| Provide fix or mitigation | **≤ 14 days** |
| Public disclosure (if agreed) | After fix is deployed |

---

## 🏅 Recognition

Every valid report that leads to a fix will be credited:

- In the relevant commit or pull request message
- In the repository's acknowledgements section
- Publicly credited by name/handle (with your consent)

---

## ⚖️ Safe Harbor

Acting in good faith under this policy?  
I will **not** pursue legal action against you, provided you:

- Keep the report private until resolved
- Do not exploit the vulnerability beyond confirming it exists
- Do not access, copy, modify, or delete data that isn't yours
- Follow the spirit of responsible disclosure
