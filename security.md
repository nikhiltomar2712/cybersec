# Security Policy

## Overview

This repository is a **personal learning and notes** resource focused on cybersecurity concepts, tools, and techniques. It is maintained for educational purposes only.

If you discover any security issues within this repository itself — such as accidental credential leaks, malicious code, or exposed sensitive data — please follow the responsible disclosure process outlined below.

---

## Supported Scope

Since this is a learning/notes repository, the following applies:

| Area | In Scope |
|------|----------|
| Exposed API keys, tokens, or credentials in any file | ✅ Yes |
| Malicious or unsafe code snippets committed by mistake | ✅ Yes |
| Broken or dangerous external links | ✅ Yes |
| Typos or incorrect technical information | ⚠️ Use Issues tab |
| Suggestions for new topics or improvements | ⚠️ Use Issues tab |

## Reporting a Vulnerability

**Please do NOT open a public GitHub Issue for security vulnerabilities.**

Instead, report it privately using one of the methods below:

### Option 1 — GitHub Private Security Advisory (Preferred)

1. Go to the **Security** tab of this repository
2. Click **"Report a vulnerability"**
3. Fill in the details and submit

GitHub will keep the report confidential until it is resolved.

### Option 2 — Email

Send a detailed report to:

```
nikhiltomar2712@gmail.com
```

Use the subject line: `[SECURITY] <brief description>`

---

## What to Include in Your Report

A good vulnerability report helps resolve issues faster. Please include:

- **Description** — What is the issue and where did you find it?
- **Steps to reproduce** — How can it be confirmed?
- **Potential impact** — What could an attacker do with this?
- **Suggested fix** (optional) — Any ideas on how to resolve it?

---

## Ethical Use Notice

All content in this repository is intended **strictly for educational and defensive security research**. The tools, techniques, and notes documented here must only be used:

- On systems you own
- On systems you have **explicit written permission** to test
- In controlled lab environments (e.g., TryHackMe, HackTheBox, local VMs)

Unauthorized use of any technique described here against systems you do not own is **illegal and unethical**. The repository maintainer holds no responsibility for misuse.

---

## Out of Scope

The following will **not** be treated as security issues:

- Publicly known CVEs referenced for educational discussion
- Intentionally vulnerable lab environments (e.g., DVWA, Metasploitable)
- Theoretical attack explanations without proof-of-concept targeting real systems
